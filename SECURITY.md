# Security Policy

`better-auth-rs` is security-critical software. We appreciate responsible disclosure.

## Reporting a vulnerability

**Do not open a public issue for security vulnerabilities.** Instead, report privately via
[GitHub Security Advisories](https://github.com/rubentalstra/better-auth-rs/security/advisories/new)
(Security → Report a vulnerability).

Please include: affected version/commit, a description and impact assessment, and a minimal
reproduction if possible. We aim to acknowledge within a few days and will credit reporters unless
they prefer otherwise.

## Scope

This is an early-stage port under active development. Until a stable `1.0`, treat it as
not-yet-audited and avoid relying on it for production authentication.

## Upstream

Vulnerabilities in the underlying design may also affect
[better-auth](https://github.com/better-auth/better-auth); we coordinate with upstream where
relevant.
